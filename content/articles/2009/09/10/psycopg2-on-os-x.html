--- kind: article created_at: 2009-09-10 12:02:15 +01:00 title: psycopg2 on OS X tags: - Apple - Developing - easy_install - enterprisedb - OS X - pip - postgresql - psycopg2 - Python --- I'm still getting used to the quirks of developing with Python on OS X (I really need to finish my "Why I switched" post...), and the latest thing to trip me up was a) installing Python modules and b) install psycopg2 in particular. Regarding a), I've now been enlightened after discovering the joys of virtualenv (+ virtualenvwrapper of course), easy_install and now pip. Now pip - being the newer of the two and being virtualenv aware - is obviously better, but you still have to use easy_install to get it in the first place. virtualenv makes managing multiple projects with multiple versions of libraries (i.e. Django) sane, but overall I still get twitchy when installing applications - I miss apt-get and Synaptic! Yes, I know about Fink etc., but so far the experience with those has been suboptimal. For PostgreSQL, I went with the installer for 8.4 maintained over at EnterpriseDB. However when I tried to install psycopg2, I got an unhelpful message from both easy_install and pip about a file not being found, but not which file. A quick Google around led my to this post which pointed me in the right direction. The trick is to provide pip (or easy_install) with the bin path for PostgreSQL. Therefore the magic incantation for me was: 
$ PATH=$PATH:/Library/PostgreSQL/8.4/bin/ sudo pip install psycopg2
As an aside, I've been using sudo for years (and years), so why do I keep forgetting to use it with OS X? All working now!
